用 Webmention 连接 Mastodon 并通过 Github Actions 拉取更新

06 January 2024

Rongbin Fan

变相等于用长毛象作为你 Hugo 的评论系统，并加入联邦宇宙。

什么是 webmention #

Webmention是由IndieWeb发起的W3C标准，用来实现网页之间的相互提及，不必借助平台或引入评论系统。它的工作流程是这样的：

  1. 你在自己的网站上传了一篇文章
  2. 这篇文章被我在自己的网站里引用
  3. Webmention服务确认了我的引用
  4. 你可以通过API、Atom Feed等方式取得通知

Webmention可以借助webmention.io服务或自行挂载，这里给出使用webmention.io的步骤：

1. 用IndieAuth标识你的身份 #

Webmention.io使用IndieAuth进行登录，所以需要先设置IndieAuth。在index.html中，对自己的帐号说明为rel="me"，例如：

<a rel="me" href="mailto:your_email@email.com">Send me an email</a>

或者以隐藏的方式在<head>说明：

<link rel="me" href="mailto:your_email@email.com">

这样一来，采用IndieAuth登录的地方，会通过这些方式验证你的身份，例如通过向邮箱发送邮件，或跳转Github登录。

2. 使用 webmention.io 挂载 #

在webmention.io登录后，你会看到一个Settings-Setup页面，其中给出了需要你在<head>加入的tag：

<link rel="webmention" href="https://webmention.io/your_domain/webmention" />

这条tag说明你的页面能够接受别人的提及，你可以通过webmention.rocks测试能否接收到。

你可以通过webmention.io提供的Mentions Feed，在其他人提及时获得通知。

3. 借助 webmention.js 在博文页面显示 webmention #

Webmention.io提供API返回别人提及的具体内容，你可以通过Fetch API的方式自定。这里我们借助前人造好的轮子webmention.js，根据readme，在你的网站存放webmention.min.js文件并用<script src=""></script>引入，在你需要展示webmention列表的地方插入：

<div id="webmentions"></div>

你可以通过新建一个测试页面，用来提及目标页面，以测试是否能正常发出和显示。你也可以等到后面完成连接Mastodon之后，通过发toot和回复的方式测试。webmention列表的样式可通过css自定义。

4. 连接到 Bridgy #

Bridgy提供的服务可将网站和社交平台相联，没有个人页面的用户，也能通过支持的平台进行回复，依靠Bridgy进行webmention。进入主页选择Mastodon，使用cross-post或直接连接联邦宇宙的方式均可，我使用cross-post。

这样一来，任何包括了你博文url的toot，对toot的回复、转发、喜爱，都会作为webmention传到你的页面。

5. 使用 Github Pages 自动上传/更新博文 #

由于Hugo是静态的，在每次上传或修改.md博文时，需要通过curl或类似Telegraph的服务向https://fed.brid.gy发出对应的webmention：

curl -i -d source=https://your-domain.com/your-post/ -d target=https://fed.brid.gy https://fed.brid.gy/webmention

但我希望实现根据我是否上传或更新.md，自动发出webmention。我发现tj-actions/changed-files能够找到repo下文件的修改情况，通过寻找是否有文件被修改，之后对应用curl更新即可。

在之前的博文里，我设置了deploy.yml用来自动部署Hugo、生成静态文件。于是对应在这里设置为只在完成deploy后才运行。

name: send webmentions according to md changes

on:
  workflow_run:
    workflows: ["deploy"]
    types: 
      - completed
    branches:
      - main

jobs:
  changed_files_send_webmention:
    runs-on: ubuntu-latest
    name: send webmentions according to md changes
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: get changed files
        id: changed-files
        uses: tj-actions/changed-files@v41
        with:
          files: content/posts/*.md # 注意这里根据实际需要修改目录位置

      - name: list all changed files
        run: |
          ALL_CHANGED_FILES="${{ steps.changed-files.outputs.all_changed_files }}"
          for file in $ALL_CHANGED_FILES; do
            echo "$file was changed"
          done          

      - name: send webmentions
        if: steps.changed-files.outputs.all_changed_files != '' # 判断不为空才运行
        run: |
          CHANGED_FILES=$(echo "${{ steps.changed-files.outputs.all_changed_files }}" | tr ',' '\n')
          for FILE in $CHANGED_FILES; do
            if [[ $FILE == content/posts/*.md ]]; then
              POST_NAME=$(basename "$FILE" .md)
              POST_URL="https://fanrongbin.com/${POST_NAME}" # 注意修改为你的网站域名
              echo "Sending Webmention for $POST_URL"
              curl -i -d source=${POST_URL} -d target=https://fed.brid.gy https://fed.brid.gy/webmention # 如果你使用的不是 Bridgy，请对应修改 target url
            fi
          done

测试是否能够发出webmentions。
